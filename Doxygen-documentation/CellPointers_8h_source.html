46 // The members below are being untangled into two conceptual groups as part of
47 // the WorksheetDocument / view split (see the "Document-side" and "View /
48 // interaction-side" banners further down). Document-side pointers describe the
49 // document model (what is selected, which cell is active/answering, what maxima
50 // is working on, which cells hold errors). View/interaction-side pointers
51 // describe transient window state (hover, drag/keyboard-selection anchors,
52 // scroll targets, animation timers). Both halves must remain reachable through
53 // the Configuration -> CellPointers registry because cells read and write them
54 // via Cell::GetCellPointers(); the split is therefore an *internal* reorg, not
55 // a relocation of these members onto another owner. Members are moved behind
56 // accessors one self-contained concern at a time; until then most stay public.
